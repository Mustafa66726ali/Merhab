# بوت واتساب — اختبار أتمتة مرحّاب

بوت محلي لإرسال الدعوات والتذكيرات تلقائياً في **بيئة التطوير** دون تدخّل يدوي،
ويحاكي سلوك الإنسان (تأخير عشوائي، مؤشّر كتابة، فترات راحة، حدّ إرسال بالساعة)
لتقليل احتمال الحظر. **في الإنتاج تُستبدل هذه الطبقة بـ Twilio** عبر ضبط
`WHATSAPP_PROVIDER=api` وتكوين تكامل Twilio من لوحة التكاملات.

> ملاحظة: استخدام أتمتة WhatsApp Web غير رسمي وقد يُعرّض الرقم للحظر. استخدمه
> لأغراض الاختبار فقط وبرقم مخصّص للتجربة.

## التشغيل

```bash
cd whatsapp-bot
npm install
npm start
```

عند أول تشغيل سيظهر رمز QR في الطرفية — افتح واتساب على هاتفك ← **الأجهزة
المرتبطة** ← **ربط جهاز** وامسح الرمز. تُحفظ الجلسة محلياً في `.wwebjs_auth/`
فلا تحتاج لإعادة المسح في كل مرة.

## ربطه بالـ Backend

في ملف `backend/.env` (أو متغيّرات البيئة) فعّل وضع البوت:

```env
WHATSAPP_PROVIDER=bot
WHATSAPP_BOT_URL=http://127.0.0.1:8088
WHATSAPP_BOT_TOKEN=merhab-bot-dev
```

ثم من لوحة الدعوات فعّل خيار **«إرسال تلقائي»** قبل الإرسال؛ سيتولّى الباك-إند
تمرير كل رسالة إلى البوت الذي يضعها في طابور ويُرسلها بتأنٍّ بشري.

## واجهة HTTP (محميّة برمز Bearer)

| الطريقة | المسار     | الوصف                                  |
| ------- | ---------- | -------------------------------------- |
| GET     | `/status`  | جاهزية الاتصال، حجم الطابور، الإحصاءات |
| GET     | `/queue`   | حجم الطابور الحالي                      |
| POST    | `/send`    | `{ "to": "9665...", "message": "..." }` |

## ضبط سلوك المحاكاة (اختياري)

تُضبط عبر متغيّرات البيئة قبل التشغيل:

| المتغيّر          | الافتراضي | الوصف                                  |
| ----------------- | --------- | -------------------------------------- |
| `PORT`            | `8088`    | منفذ واجهة HTTP                         |
| `BOT_TOKEN`       | `merhab-bot-dev` | رمز الحماية                     |
| `MIN_GAP_MS`/`MAX_GAP_MS` | `9000`/`23000` | التأخير بين الرسائل          |
| `MAX_PER_HOUR`    | `50`      | الحدّ الأقصى للرسائل في الساعة          |
| `BREAK_EVERY_MIN`/`MAX` | `14`/`24` | عدد الرسائل قبل فترة راحة         |
| `BREAK_MS_MIN`/`MAX` | `60000`/`180000` | مدة فترة الراحة             |

## الإنتاج (Twilio)

1. كوّن تكامل **WhatsApp (Twilio)** من لوحة التكاملات (Account SID + Auth Token + رقم المُرسِل).
2. اضبط `WHATSAPP_PROVIDER=api`.
3. لن يُستخدم البوت؛ سيمرّ الإرسال عبر Twilio API مباشرة.
